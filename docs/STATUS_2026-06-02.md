## What's blocking what

### Blocking: GitHub Actions enablement

- **Symptom**: `gh api repos/hawcx/hx_agentic_sdk/actions/permissions` returns HTTP 403. Cannot dispatch workflows.
- **Resolver**: repo admin (`skhawcx` or `vishwap-commits`) checks https://github.com/hawcx/hx_agentic_sdk/settings/actions:
  - "Actions permissions" → "Allow all actions and reusable workflows"
  - "Workflow permissions" → "Read and write permissions"
- **Worth also checking**: org-level Actions policy at https://github.com/organizations/hawcx/settings/actions — same setting may need to be permissive at the org level.
- **Downstream impact**:
  - First release dispatch cannot run.
  - First `v0.1.0-alpha.1` release cannot be produced.
  - Mechanism 2 CI cannot be validated end-to-end.

### Not blocking, but worth fixing post-alpha

- `cargo package -p haap-rsv` fails because path-dep `haap-core` (in hx_labs) lacks a `version` field. Two paths to unblock:
  1. Add `version` to hx_labs's `haap-core`/`haap-crypto`/`haap-wire`/`haap-ipc` package metadata + publish those crates first. Preferred long-term.
  2. Vendor the minimal cascade-call surface into the SDK so the publishable crate has no path-dep. Rejects the "thin wrapper" principle.
- Decision deferred. `publish = true` on `haap-rsv` reflects intent; manual `cargo publish` remains blocked.
